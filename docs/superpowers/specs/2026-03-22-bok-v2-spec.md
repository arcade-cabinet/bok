# Bok v2: The Builder's Tome — Game Design Specification

**Date:** 2026-03-22
**Status:** Active

## 1. Game Identity

**Bok: The Builder's Tome** is a voxel exploration-adventure with infinite procedurally generated worlds. Players explore, build, fight, and discover at their own pace across many unique islands, each with its own character.

**Genre:** Open-world voxel adventure with roguelike progression
**Art Style:** CubeWorld voxel aesthetic — cohesive, colorful, readable
**Platforms:** Web, iOS, Android (via Capacitor)

## 2. New Game

When the player starts a new game, they see a simple diegetic prompt:

- **Seed** — text input or shuffle button (determines ALL world generation)
- **Mode** — Survival or Creative (locked for the life of this save)

That's it. No biome selection. No complex modal. The seed and mode are stored in SQLite and never change. This creates a unique game world.

Once confirmed, the player enters the **Hub Island**.

## 3. The Hub Island

The hub is a **fixed, hand-crafted tropical island** — not procedurally generated. It is the SAME hub for every game, regardless of seed. It is beautiful, warm, inviting. The player's permanent home.

### 3.1 Layout

The hub is a lush tropical island with:
- **Central area** — player spawn, open grassy clearing
- **Buildings** — Armory, Library, Forge, Market (upgradeable via resources)
- **NPCs** — Blacksmith, Scholar, Shopkeeper, Navigator (interact for services)
- **Docks** — the KEY feature

### 3.2 The Docks

The docks are **actual wooden block docks** extending out from the shoreline. Each island biome has its OWN dock with:

- **A wooden pier** built from wood plank blocks extending into the water
- **A procedurally generated signpost** at the end of each dock
- **The island name** drawn/rendered on the signpost (e.g., "Whispering Woods")
- **A colored banner** matching the island's biome theme

For **unlocked islands** (Survival mode): The signpost shows the island name and the player can walk onto the dock and "Set Sail" to travel there.

For **locked islands** (Survival mode): The signpost has a **"Come Back Later" sign** — a piece of paper/parchment pinned to the signboard. The dock still exists but the player can't sail from it.

In **Creative mode**: ALL docks are accessible from the start.

The **Forest dock** is always the first one unlocked. It's closest to the player spawn. Other docks extend further along the shoreline.

### 3.3 Diegetic Design

Everything is IN the world. No menus for island selection. The player physically walks to a dock, reads the signpost, and sets sail. This is proper diegetic design — the world IS the UI.

## 4. Islands

Each island is its own **infinite procedurally generated world** with a unique identity.

### 4.1 Island Generation

Each island is generated from: `seed + biomeId`

This combination produces a deterministic, unique infinite world:
- Terrain shape (amplitude, frequency, base height, water level)
- Block palette (unique combination of block types and shapes for THIS island)
- Vegetation and decoration placement
- Enemy spawn patterns
- Structure placement (ruins, dungeons, shrines)
- Boss arena location

### 4.2 Unique Feel Per Island

Each island must feel DISTINCT. This comes from:

1. **Unique block combinations** — not just "this biome uses sand." Each island uses the seed to select a SPECIFIC combination of blocks, shapes (slabs, ramps, stairs, poles), and textures that make it unique even among islands of the same biome type.

2. **Unique terrain profile** — the seed affects noise parameters, creating different hills, valleys, caves, and features.

3. **Unique structures** — the seed places ruins, dungeons, shrines, and towers at different locations with different templates.

4. **Unique enemy composition** — while biome determines the enemy POOL, the seed affects spawn density, patrol routes, and elite/variant placement.

5. **Unique goals** — each island has its own set of goals themed to its personality.

### 4.3 Eight Island Biomes

| Biome | Character | Key Blocks |
|-------|-----------|------------|
| Forest | Dense, verdant, mysterious | Grass, wood, leaves, moss |
| Desert | Vast, ancient, scorching | Sand, sandstone, desert stone |
| Tundra | Harsh, pristine, isolated | Snow, ice, frozen dirt, permafrost |
| Volcanic | Dangerous, dramatic, fiery | Obsidian, ash, basalt, lava, magma |
| Swamp | Murky, treacherous, hidden | Mud, peat, swamp stone, dead wood |
| Crystal Caves | Magical, glowing, otherworldly | Crystal, geode, amethyst, quartz |
| Sky Ruins | Floating, ancient, vertiginous | Cloud, sky stone, wind block |
| Deep Ocean | Dark, alien, crushing | Coral, sea floor, deep water, kelp |

### 4.4 Persistence

When the player visits an island:
- Block modifications are saved as chunk deltas to SQLite
- Goal progress is saved per-island
- Killed enemies and opened chests are tracked
- When the player RETURNS, everything is exactly as they left it

## 5. Goals & Progression (Survival Mode)

### 5.1 Per-Island Goals

Each island has 3 goals that drive exploration:
1. **Discover** — find landmarks (shrines, dungeon entrances)
2. **Defeat** — kill enough enemies
3. **Collect** — open treasure chests

Goals are themed to each island's personality. They push the player to explore but don't PUNISH builders or explorers — you can ignore goals and just build.

### 5.2 Boss Gate

Completing all 3 goals on an island unlocks the boss fight. The boss is somewhere in the world — find it and fight it.

### 5.3 Progression Chain

Defeating a boss:
1. Grants a **Tome page** (permanent ability)
2. Unlocks the NEXT island's dock on the hub
3. The "Come Back Later" sign is removed from the next dock's signpost

### 5.4 Creative Mode

No goals. No boss gate. All docks accessible. Build and explore freely.

## 6. Building System

### 6.1 Block Placement

The player can place and break blocks in the world. This is a CORE mechanic, not an afterthought.

**Placing blocks:**
- Player selects a block type from their inventory hotbar
- Right-click/tap places a block on the face of an adjacent block
- Preview ghost shows where the block will go before placing
- ALL block shapes available: cube, slabBottom, slabTop, poleY, ramp, rampCornerInner, rampCornerOuter, stair, stairCornerInner, stairCornerOuter
- Shape selected via hotbar sub-menu or cycle key

**Breaking blocks:**
- Left-click/hold to break (tool-dependent speed)
- Wood tool = slow, Stone = medium, Gold = fast, Diamond = fastest
- Broken blocks drop as collectible resources
- Some blocks require specific tools (stone needs pickaxe, wood needs axe)

**Block palette per island:**
Each island's biome determines which blocks are available to find AND place:
- Forest: grass, dirt, stone, wood, leaves, moss
- Desert: sand, sandstone, desert stone, oasis water
- Volcanic: obsidian, ash, basalt, magma crust
- etc.

Players can bring resources BETWEEN islands via their inventory. Build a desert-themed house on the forest island if you want.

### 6.2 Building Persistence

All block modifications are saved as chunk deltas to SQLite per-island. When the player returns to an island, their builds are exactly as they left them. This is the "Minecraft promise" — your creations persist.

### 6.3 Block Shapes for Creative Building

The VoxelRenderer supports 11+ block shapes. This gives builders real architectural tools:

| Shape | Use Case |
|-------|----------|
| Cube | Standard building block |
| Slab (bottom/top) | Half-height floors, ceilings, shelves |
| Pole (vertical) | Fences, pillars, supports |
| Ramp | Slopes, roofs, terrain smoothing |
| Ramp Corner (inner/outer) | Curved roof corners, natural slopes |
| Stair | Staircases, stepped terrain |
| Stair Corner (inner/outer) | Spiral staircases, L-shaped steps |

Shape selection via hotbar cycle or dedicated UI. Each shape can use any block texture.

## 7. Island Resources & Crafting

### 7.1 Per-Island Resources

Each island has its own set of harvestable resources tied to its biome:

| Island | Exclusive Resources | Harvest Method |
|--------|-------------------|----------------|
| Forest | Wood, Leaves, Herbs, Mushrooms | Axe (trees), Hand (plants) |
| Desert | Sand, Sandstone, Ancient Bone, Cactus Fiber | Pickaxe, Hand |
| Tundra | Ice, Frost Shard, Snow, Permafrost | Pickaxe |
| Volcanic | Obsidian, Magma Crystal, Ash, Basalt | Diamond Pickaxe |
| Swamp | Peat, Dead Wood, Mushroom, Poison Gland | Axe, Hand |
| Crystal | Amethyst, Quartz, Geode, Crystal Shard | Pickaxe |
| Sky Ruins | Cloud Block, Sky Stone, Wind Essence | Pickaxe |
| Deep Ocean | Coral, Kelp, Pearl, Void Essence | Hand |

Common resources (stone, dirt, coal) are found everywhere.

### 7.2 Crafting System

Crafting is done at the **Forge** building on the hub island:
- Recipes require resources from specific islands
- This creates MOTIVATION to visit different islands and gather
- Higher-tier recipes require resources from harder islands

Example recipes:
- Iron Sword = 10 Iron Ore (any island) + 5 Wood (Forest)
- Fire Staff = 5 Magma Crystal (Volcanic) + 3 Wood
- Ice Wand = 5 Frost Shard (Tundra) + 3 Crystal Shard (Crystal)
- Diamond Pickaxe = 3 Diamond (rare, any island) + 2 Obsidian (Volcanic)

### 7.3 Resource Goals

Some island goals are resource-based:
- "Gather 10 Frost Shards" (Tundra)
- "Harvest 5 Ancient Bones" (Desert)
- "Collect 3 Void Essence" (Deep Ocean)

This ties resource gathering into the progression system naturally.

## 8. Combat

### 6.1 Player
- Weapon combos (3-hit chains)
- Dodge (i-frames, stamina cost)
- Block (hold, 50% reduction) / Parry (first 0.15s, full negate)
- Weapon tiers: Wood → Stone → Gold → Diamond
- Stamina system (100 max, 20/sec regen)

### 6.2 Enemy AI (Yuka GOAP)
- Melee: chase + attack (skeleton, goblin, zombie)
- Ranged: keep distance + cast (wizard, archer)
- Pack: coordinated chase (wolf)
- Ambush: hide + surprise (swamp-lurker)
- Each enemy biome-tinted for visual variety

### 6.3 Boss AI (Yuka CompositeGoal)
- 3 phases at health thresholds (100/66/33%)
- Phase-specific evaluator biases
- Attacks from content JSON (melee, ranged, AoE, summon)

## 7. Technical Architecture

### 7.1 Rendering — JollyPixel Engine
- VoxelRenderer for terrain (chunk-based, tileset atlas)
- ModelRenderer for 3D models (enemies, props, NPCs)
- AssetManager for batch preloading
- Camera3DControls for FPS camera
- ParticleSystem, WeatherSystem, DayNightCycle, LODManager

### 7.2 State — Koota ECS
- All game entities are Koota world spawns
- Traits define data, systems define behavior
- frameloop.ts orchestrates system execution

### 7.3 AI — Yuka
- Vehicle + steering for enemy movement
- Think + GoalEvaluator for GOAP decisions
- CompositeGoal for boss phase trees

### 7.4 Persistence — SQLite (Capacitor)
- Game save: seed + mode (immutable)
- Per-island: chunk deltas, goal progress, enemy/chest state
- Progression: tome pages, building levels, run history

### 7.5 Assets — CubeWorld + Voxel Packs
237 glTF models across 11 categories, all registered in models.ts.

## 8. Testing Standards

- Tests describe what the game SHOULD do, not what it currently does
- Write the test FIRST, then build to pass it
- Browser tests for every visual component
- Playtest on real devices before claiming done
- Every asset in public/assets/ must be code-referenced and spawned
