# Bok v2: The Builder's Tome — Revised Game Design Specification

**Date:** 2026-03-22
**Status:** Active
**Supersedes:** 2026-03-20 spec (island-based roguelike → infinite world)

## 1. Game Identity

**Bok: The Builder's Tome** is a voxel exploration-adventure with infinite procedurally generated worlds. Players explore, build, fight, and discover at their own pace. The game respects all play styles — builders, explorers, and combat enthusiasts.

**Genre:** Open-world voxel adventure with roguelike elements
**Art Style:** CubeWorld voxel aesthetic — cohesive, colorful, readable
**Platforms:** Web, iOS, Android (via Capacitor)
**Session Length:** 5 minutes to hours — player decides
**Multiplayer:** Single-player

## 2. Game Modes

### 2.1 Creative Mode
- All biomes unlocked from start
- Player is invincible (no damage taken)
- Infinite resources
- Build and explore freely
- No goals or boss gates
- Passive animals spawn (no hostile enemies)

### 2.2 Survival Mode
- Start with forest biome only
- Combat with consequences (death loses per-run gear)
- Goal-based progression unlocks boss fights
- Defeating bosses unlocks new biomes
- Permanent unlocks: Tome pages (abilities)
- Per-run gear lost on death

## 3. World Generation

### 3.1 Many Infinite Worlds
Each island is its own **infinite world** with a unique seed + biome combination. Players sail between islands. Each island:
- Has its own seed-based terrain generation
- Has its own biome (terrain, blocks, weather, sky, enemies)
- Has its own goals (unique per island)
- Has its own saved block modifications (deltas in SQLite)
- Has its own boss to discover and defeat
- Can be RETURNED TO — builds and changes persist

The hub is also its own infinite world with a fixed seed.

### 3.2 Chunk System
- 16-block chunks load/unload around player
- Render distance: 4 chunks (64 blocks) on desktop, 3 on mobile
- Same seed + biome = same world, always
- No map edges — explore forever in any direction

### 3.2 Biome Regions
Each biome is an infinite region with unique:
- **Terrain shape** — amplitude, frequency, base height, water level
- **Block palette** — surface, subsurface, stone, water, accent blocks
- **Block shapes** — slabs, ramps, stairs, poles for variety (not just cubes)
- **Sky/fog colors** — atmosphere matching biome theme
- **Weather** — sandstorms, blizzards, volcanic embers, swamp fog
- **Vegetation** — trees, flowers, cacti, crystals, dead wood, kelp
- **Structures** — ruins, dungeons, shrines, market stalls, watchtowers
- **Enemies** — biome-specific with color tinting
- **Animals** — biome-appropriate passive mobs
- **Music** — per-biome exploration track

### 3.3 Eight Biomes
| Biome | Terrain | Unique Feature | Weather |
|-------|---------|----------------|---------|
| Forest | Rolling hills, dense trees | Limited visibility canopy | None |
| Desert | Flat dunes, sparse | Sandstorms (periodic) | Tan sandstorm |
| Tundra | Gentle snow, ice lakes | Freezing cold | White blizzard |
| Volcanic | Dramatic peaks, lava | Rising lava danger | Orange embers |
| Swamp | Flat, murky water | Ambush enemy spawns | Green fog |
| Crystal Caves | Medium, crystal pillars | Light puzzles | None |
| Sky Ruins | Extreme vertical, gaps | Falling + wind | None |
| Deep Ocean | Below sea level | Underwater combat | None |

### 3.4 Persistence
- Block modifications saved as chunk deltas to SQLite
- Seed + deltas = full world state
- Saves tied to seed — loading a game = loading its seed + deltas
- Run history and progression persisted separately

## 4. Structures (3D Model Placement)

Structures are placed deterministically by the procedural generator using converted voxel assets:

- **Stone ruins** — walls, pillars, broken pieces (Stone/ models)
- **Dungeon entrances** — gates, torches, dark pillars
- **Shrines** — 4 pillars + flower (goal landmark targets)
- **Market stalls** — barrels, crates, stall structures (Objects/ models)
- **Watchtowers** — stacked stone walls + torch
- **Village houses** — modular wall/roof/corner pieces (Village/ models)

## 5. Goal System (Survival Mode)

Each biome has 3 goals that drive exploration:
1. **Discover** landmarks (shrines, dungeon entrances) — walk near them
2. **Defeat** enemies — kill count in this biome
3. **Collect** loot — open chests

Completing all 3 goals unlocks the boss fight for that biome. Goals are tracked in a HUD overlay. Goals are optional — the world is always explorable without completing them.

## 6. Combat

### 6.1 Player Combat
- **Weapon combos** — 3-hit chains with damage multipliers
- **Dodge** — i-frames (0.3s), stamina cost (25)
- **Block** — hold for 50% damage reduction
- **Parry** — first 0.15s of block = full damage negate
- **Stamina** — 100 max, 20/sec regen, limits dodge/sprint
- **Weapon tiers** — Wood → Stone → Gold → Diamond (CubeWorld tools)

### 6.2 Enemy AI (Yuka GOAP)
Each enemy type has a Think brain with goal evaluators:
- **Melee** (skeleton, goblin, zombie) — chase + attack
- **Ranged** (wizard, skeleton-archer) — keep distance + cast
- **Pack** (wolf) — group chase, coordinated
- **Ambush** (swamp-lurker) — hide + surprise attack

Enemies are biome-tinted (same model, different color per biome).

### 6.3 Boss AI (Yuka CompositeGoal)
Each boss has a phase-aware brain:
- Phase 1 (100-66% HP): Strategic — circle strafe, measured attacks
- Phase 2 (66-33% HP): Aggressive — summons, retreat + barrage
- Phase 3 (33-0% HP): Desperate — enrage, all-out melee

5 evaluators shift desirability per phase. Attacks come from content JSON.

## 7. Entities (Koota ECS)

All game entities are Koota world spawns with traits:

### 7.1 Entity Types
- **Player** — IsPlayer, Transform, Movement, Health, Stamina, WeaponStats
- **Enemy** — IsEnemy, Transform, Movement, Health, ContactDamage, AIState, YukaRef, Ref
- **Boss** — IsEnemy + IsBoss, phase-aware brain
- **Animal** — IsAnimal, Transform, Movement, AIState (wander)
- **Chest** — IsChest, ChestState, Interactable
- **Prop** — IsProp, Transform, Ref (trees, rocks, crystals)
- **Tool** — IsTool, WeaponStats
- **NPC** — IsNPC, Interactable

### 7.2 Systems (one per file, (world: World) => void)
- update-time, update-movement, update-enemy-ai, update-combat
- spawn-enemies, update-goals, sync-rendering, update-chunks
- spawn-props, spawn-animals, spawn-structures

## 8. Assets (CubeWorld + Voxel Packs)

All assets in `public/assets/` using glTF format:
- **Animals/** — 26 models (CubeWorld + voxel pack)
- **Blocks/** — 15 CubeWorld block models
- **Characters/** — 4 player models
- **Collectibles/** — 11 food/resource items
- **Enemies/** — 9 CubeWorld enemy models
- **Environment/** — 49 decoration models
- **Objects/** — 28 voxel RPG objects (barrels, torches, etc.)
- **Pixel Blocks/** — 18 pixel-art blocks
- **Stone/** — 42 dungeon/ruin building pieces
- **Tools/** — 16 weapon/tool models (4 tiers)
- **Village/** — 19 modular house pieces
- **Atlas.png** — CubeWorld texture atlas

Total: **237 glTF models**, all registered in `models.ts`.

## 9. Rendering (JollyPixel Engine)

- **VoxelRenderer** — chunk-based terrain with tileset atlas
- **ModelRenderer** — 3D model loading via JollyPixel actors
- **AssetManager** — batch preloading with deduplication
- **Camera3DControls** — FPS camera with smooth acceleration
- **ParticleSystem** — hit bursts, death effects, ambient
- **WeatherSystem** — biome-specific particle weather
- **DayNightCycle** — 240-second cycle with biome tinting
- **LODManager** — distance-based culling for performance

## 10. Audio (Tone.js + JollyPixel)

- Per-biome music tracks (9 OGG files)
- Procedural combat SFX (Tone.js synthesis)
- Atmospheric ambient SFX (8 OGG files)
- Spatial audio (Panner3D for 3D positioning)
- Audio compressor (prevents clipping)
- Mobile autoplay resume on first gesture

## 11. Platform

- **Web** — PWA with service worker, offline caching
- **iOS/Android** — Capacitor with SQLite, haptics, orientation lock
- **CI/CD** — GitHub Actions: typecheck, lint, unit tests, browser tests, web build, Android APK
- **Deploy** — GitHub Pages auto-deploy on merge to main

## 12. Testing

- **Unit tests** — Vitest, all game logic
- **Browser tests** — Vitest browser mode with Playwright
- **E2E** — Maestro flows for web + iOS Safari
- **Content validation** — Zod schemas for all JSON
- **Balance tests** — stat ranges for enemies/bosses/weapons

Tests describe what the game SHOULD do, not what it currently does.
